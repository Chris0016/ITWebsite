Christopher PerezJune 19, 2021 at 6:30 pm

Configuring the BIOS/UEFI Settings

 Although not a common source of troubleshooting, the BIOS Interface/UEFI can be a great way to fix or personalize your computer system. The BIOS itself is essentially firmware that connect external devices like the keyboard, mice, or memory to a computer system. The BIOS Interface contains many settings for the hardware in a system. These settings are accessed by using a designated key during startup or navigating through the settings in MS Windows. The UEFI is an upgrade from the BIOS and is the common standard in current systems. The UEFI has far more capabilities than BIOS along with an easier to use graphical interface that supports a mouse. This article will take a technical approach to configuring the BIOS/UEFI settings, but if you'd like to learn more about the nature of the BIOS/UEFI you can read our article here.

Basic Settings

 The BIOS settings interface/UEFI(I'll refer to it as UEFI) enables users to make adjustments to hardware settings. Within the BIOS settings interface there is also the BIOS component information(CMOS Setup Program) that deals with 'foreign' devices. There are some simple settings that even novice users can dabble with to configure their system.

Basic Settings: Hard Drive

 Apart from the settings you can also view system details in the BIOS. While most devices today implement a technology called plug and play, older BIOSes give the option to manually input specs about a device such as the hard drive. You can input the dimensions found on the back of the hard drive. Meanwhile, the system will the size of the drive is calculated automatically. Other systems determine size by the 'drive type'. Again, you can find this information on the drive itself.

Basic Settings: Parallel Port and Serial Port

 In the modern world USB cabling has become the standard due to its speed and cost. However, other systems use parallel or serial ports. For such systems, the UEFI is a great source for troubleshooting. There are several parallel port modes available; Standard Parallel Port(SPP), Enhanced Parallel Port(EPP), and Extended Capabilities Port(ECP). If you've ever had issues with the parallel port it might be because it is in the inappropriate mode.

 SPP: Runs at 150KBps and only supports data traveling from the computer the device.
 EPP: Runs at 2MBps and supports bidirectional communication as well as daisy chaining.
 ECP: Runs at speeds greater then 2 MBps and supports bidirectional communication.
 If your system has serial ports, chances are that your BIOS has configurations for these ports. For serial ports, the BIOS gives the option to input a custom 'I/O address' and 'interrupt request'(IRQ).
 You can also enable/disable other types of ports in the UEFI like USB.

Basic Settings: Boot Options and Sequence

 The software for the operating system is found on a long term storage medium like the hard drive and is then loaded into the RAM during boot. There are other options from where to load the operating system such as a non-primary drive, a thumb drive, or off a network. The CMOS stores a list of the order to search for an operating system called the 'boot sequence'. The computer system will search in chronological order for a valid operating system and boot from that source. You can rearrange the boot sequence in the UEFI or block the booting from types of devices like a thumb drive. In an environment where security is primordial, disabling boot from thumb drives is a common way from preventing users from being able to snatch confidential data.

 If you'd like to configure your system to boot off a network there are several configurations that need to be in place. The first being, your network interface card(NIC) must be enabled. The NIC acts as a radio that receives and decodes radio waves. By default the NIC is enabled, but if you are not able to view available WiFi networks is because it isn't. The next configuration that has to be made is that booting off a network must enabled on your system. Third, setting the network in the boot sequence correctly. But before making any of these changes you should make sure that you have a 'Pre-boot Execution Environment(PXE)- compliant system'. A PXE-compliant system is simply a computer that has a BIOS that supports booting from a network.

UEFI Security

 If you are a user concerned with data security, exploring and tweaking the UEFI settings can lead you to some great preventive, detective, and recovery methods for handling your data.

UEFI Security: User and Admin password

 Setting a good password for your computer is a great first step in protecting your data. You can apply even more restrictions to your system with a user password and administrator password. . If you enable a power-on password, the user will need to input the correct key combination in order for the system to load the operating system.

 Adding an administrator password will provide a shield to the UEFI and prevent any changes from being made to the UEFI unless the correct password is entered.

 The adminstrator password is refered to as a way to secure a system at the administrative level-what the user can do. While the power-on password is a way control who can use the system.

UEFI Security: Drive Encryption

 Unfortunately, the Admin password and power-on password can be easily circumvented by resetting the BIOS. If your system does not have a physical lock, resetting the BIOS is as easy as removing the battery the CMOS is attached to(for older systems). Or if your motherboard has a BIOS jumper, you can simply set the pin in the correct combination. On modern systems it is more difficult, but still possible.

 Setting a password may protect your data from a novice hacker, but it won't protect it against dedicated intruders. Your best bet will be to implement encryption practices that take foundation on tried and true mathematics that make it very difficult for computers to decrypt data(though not impossible).

TPM

 A Trusted Platform Module(TPM) is a chip placed on the motherboard that stores the keys to decrypt a drive. Your BIOS must support TPM encryption in order for TPM be implemented. With the rollout of MS Windows 11, computers are expected to carry an integrated TPM, making devices more secure.

Secure Boot

 Secure Boot is a modern security feature fond on UEFI systems. secure boot aims to ensure that only valid software is ran during the boot up process. Fundamentally, secure boot tags software with a signature and only allows software with valid signatures to run.

 Secure Boot is enabled by default, but you can always check if it is not in the UEFI.

Virus Protection

 The Master Boot Record(MBR) is special code located the the beginning of the drive that initiates the process of loading the operating system. Certain virus attacks seek to alter the master boot record and corrupt the system. Modern systems have special virus protection embedded within the UEFI to protect against MBR attacks. This anti-virus searches for changes made to the MBR and prohibits boot if it detects a change.

 If you are trying to boot off your non-daily drive, then the built-in protection system of the UEFI will prevent you from doing so. The reason is because the MBR for the another operating system you're trying to boot from is different from the one booted from before, hence the obstruction. In windows systems, if you'd like to boot an OS other the MS Windows 7, 8, 10 or 11 from a different drive then you will need to disable virus protection in the UEFI. However, it is strongly recommended to re-enable virus protection as soon as you can.

Advanced Settings: Virtualization

 Virtualization refers to having multiple 'virtual machines' that can run their own operating system. You can think of a virtual machine as a another digital computer within your physical computer that has resources like memory and processor power allocated to it. You can enable virtualization from your settings in modern systems and it should run without trouble. If not then chances are that you need to enable virtualization in the BIOS. You can do so by traveling to 'advanced processor features' and enabling virtualization.

 Virtualization requires special software and firmware for your specific processor. Therefore, if you are unable to see virtualization options in your BIOS is because your processor does not support it.

Diagnostics and Monitoring

 The UEFI also has several built-in handy features for your managing your systems health. These features include Fan Speeds, intrusion detection, voltage measuring,CPU clock speed, and bus speed.

Diagnostics and Monitoring: Intrusion Detection

 You can detect if your system has been physically opened by enabling intrusion detection on you UEFI. Be aware that not many systems have this feature.
